# Job Aid 002 — Claude Code Permissions and CLAUDE.md Limitations

| Status   | Date       | Project Version |
|----------|------------|-----------------|
| Active   | 2026-05-09 | 1.0.1           |

## The Problem

Instructions in `CLAUDE.md` are read by Claude (the language model). Permission prompts — "Allow this bash command?", "Allow this agent?" — are enforced by the Claude Code **runtime**, not by Claude itself. Claude cannot suppress them by following an instruction. Any line in `CLAUDE.md` that says "do not ask for permission" or "the answer is always yes" has no effect on those prompts.

**What CLAUDE.md can control:** Claude's reasoning and text-generation behavior — coding style, documentation conventions, whether to ask "do you want to proceed?" before acting.

**What CLAUDE.md cannot control:** Permission prompts generated by the harness before Claude sees the tool result.

---

## The Fix — settings.json Allowlist

Permission prompts are suppressed by adding entries to the `allow` array in `settings.json`. There are three relevant settings files:

| File | Scope |
|------|-------|
| `~/.claude/settings.json` | Global — applies to all projects |
| `.claude/settings.json` | Project — committed, applies to all contributors |
| `.claude/settings.local.json` | Project-local — gitignored, personal overrides |

For personal blanket approval of all tool access, add to `~/.claude/settings.json`:

```json
{
  "permissions": {
    "allow": [
      "Bash",
      "PowerShell",
      "Agent"
    ]
  }
}
```

`"Bash"` and `"PowerShell"` without a command argument are the **tool-only form** — they match all invocations of that tool. `"Agent"` allows all subagent spawning without prompting.

Specific command patterns can also be added alongside broad rules:

```json
"allow": [
  "Bash",
  "PowerShell",
  "Agent",
  "Bash(npm run test)"
]
```

The specific entries become redundant once the broad rule is present but do not cause errors.

---

## Merging — Do Not Replace

Always read the existing file before writing. The `allow` array must be merged, not replaced. Replacing it silently drops all previously approved commands.

---

## Applying Changes

Settings are loaded at session start. After editing `settings.json`, either:

- Restart Claude Code, or
- Open `/hooks` in the UI once (forces a config reload without restarting)

---

## Reference

- Global settings: `C:\Users\<username>\.claude\settings.json`
- Project settings: `.claude/settings.json` in the repo root
- Permission rule syntax: `"ToolName"` (all invocations) or `"ToolName(pattern *)"` (prefix match)
